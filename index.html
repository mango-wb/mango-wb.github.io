环境搭建

1. 环境安装

Rust环境安装比较简单。在terminal输入:

  • $curl --proto '=https' --tlsv1.2 https://sh.rustup.rs -sSf | sh
  • 这条命令用户安装Rust的管理工具rustup。
  • 安装成功会提示：Rust is installed now. Greate!

2. 常见命令

  • 更新Rust:
    • rustup update
  • 查看已经按咋混改的工具链:
    • rustup toolchain list
  • 安装特定版本的工具链:
    • rustup toolchain install nightly # 安装nightly版本
      rustup toolchain install 1.58.0 # 安装指定版本
  • 切换到特定工具链:
    • rustup default nightly # 设置默认nightly版本
      rustup default 1.58.0 # 设置默认特定版本
  • 查看已经安装的组件:
    • rustup component list –installed
  • 安装Rust组件:
    • rustup component add rustfmt
      rustup component add clippy
  • 卸载Rust组件:
    • rustup component remove rustfmt
  • 创建新项目:
    • cargo new xxx
  • 构建项目:
    • cd xxx
      cargo build
  • 运行项目
    • cargo run
  • 运行测试：
    • cargo test
  • 发布项目
    • cargo publish

3. cargo常见命令

  • 更新依赖项
    • cargo update
  • 检查项目：
    • cargo check
  • 生成文档：
    • cargo doc –open # 生成并打开文档

4. 创建第一个rust项目

  • cargo new hello_cargo
    cd hello_cargo

  • 下边是项目目录

    • src目录用于存放rust源码文件。rust项目的代码都存放在这里。

    • target目录: cargo编译代码时生成的文件都会存放在此目录中，其中包含中间文件、可执行文件等。

    • Cargo.toml文件: 此文件是项目的配置文件，管理项目等元数据、依赖和编译选项。

      • pakage: 定义包的基本信息，如名称、版本和Rust版本
      • dependencies: 累出所需的依赖包及其版本，cargo会根据这些配置自动下载依赖项。

5. 使用cargo编译运行rust项目

  • cargo run: 此命令会在target/debug目录下生成可执行文件
  • cargo run/build --debug/--release:
    • debug模式: debug为默认模式, 生成的文件会包含调试信息，便于调试和错误追踪，使用debug进行编译未进行优化，编译速度快，但是运行速度慢。 生成的文件位于target/debug下。

      默认使用debug模式，所以会报错，不需要加–debug
    • release模式: release模式会启用编译起优化, 生成更小且更快的可执行文件，因为编译过程中编译起对其进行了优化，所以编译速度比较慢。生成的文件位于targer/release下。

6. cargo配置toml源

  • 文件位于 ~/.cargo/config.toml
    • [source.ustc]: 指定USTC的镜像地址，其中sparse+的意思表示使用Sparse Protocol是crates.io推出的协议。
